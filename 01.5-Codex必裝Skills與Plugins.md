## 四種擴充能力：先搞懂名詞

| 類型 | 白話說法 | 例子 |
|------|----------|------|
| Skills | 做事 SOP，遇到特定任務才載入 | `imagegen`、`documents`、`skill-creator` |
| Plugins | 一包工具能力，通常包含 skills 或工具 | GitHub、Browser Use、Documents |
| Connectors | 連接外部帳號或服務 | GitHub connector、Gmail、Google Calendar |
| MCP | 讓 Codex 接到外部工具 / 資料源的協定 | Obsidian MCPVault、資料庫 MCP |

> [!important]
> 新手不要追求「裝最多」。先裝最常用、最安全、最能立刻產生教學價值的組合。

## 初學者必裝清單

### 必裝 Plugins

| Plugin | 用途 | 老師使用場景 |
|--------|------|--------------|
| GitHub | 讀 repo、查 issue / PR、協助同步程式碼 | 發布教學網頁、管理懶人包 |
| Browser Use | 開網頁、測 localhost、看畫面、截圖 | 測互動教材、檢查網頁畫面 |
| Documents | 建立 / 修改 Word | 試卷、講義、研習文件 |
| Presentations | 建立 / 修改 PowerPoint | 研習簡報、教學投影片 |
| Spreadsheets | 建立 / 修改 Excel、CSV | 成績表、題庫、雙向細目表 |

### 推薦 Skills

| Skill | 用途 | 老師使用場景 |
|-------|------|--------------|
| imagegen | 生成或修改圖片 | 封面圖、教材插圖、遊戲素材 |
| openai-docs | 查 OpenAI / Codex 官方文件 | 避免照過期教學設定 |
| skill-installer | 安裝其他 skills | 擴充 Codex 能力 |
| skill-creator | 建立自己的 skills | 把重複教學流程變成 SOP |

### 需要再裝 / 進階再裝

| 項目 | 建議 |
|------|------|
| Gmail | 需要整理信件、草擬回信再連 |
| Google Calendar | 需要行程摘要、會議準備再連 |
| Obsidian / MCPVault | 第二大腦主題，建議留到後續集數 |
| Filesystem MCP | 有跨資料夾需求再處理 |
| 其他第三方 MCP | 不知道用途就先不要裝 |

## 請 Codex 幫我檢查目前已裝項目

> [!warning]
> 以下是給 Codex 讀的操作流程。請它檢查，不要一開始就要求它大量安裝。

---

## 步驟一：檢查本機 Skills

Windows 可請 Codex 執行：

```powershell
Get-ChildItem -Force -Name "$env:USERPROFILE\.codex\skills"
Get-ChildItem -Force -Name "$env:USERPROFILE\.codex\skills\.system"
```

如果看到以下項目，代表常用系統 skills 已存在：

```text
imagegen
openai-docs
skill-creator
skill-installer
plugin-creator
```

請 Codex 整理成表格：

```md
| Skill | 狀態 | 用途 |
|-------|------|------|
| imagegen | 已安裝 / 未安裝 | 生圖與改圖 |
| openai-docs | 已安裝 / 未安裝 | 查官方文件 |
| skill-installer | 已安裝 / 未安裝 | 安裝 skills |
| skill-creator | 已安裝 / 未安裝 | 建立 skills |
```

> [!tip]
> `plugin-creator` 是進階用，不是新手必學；有裝著沒關係。

---

## 步驟二：檢查 Plugins 是否啟用

請 Codex 查看設定檔：

```powershell
Get-Content -Encoding UTF8 -Path "$env:USERPROFILE\.codex\config.toml"
```

如果看到類似內容，代表 plugins 已啟用：

```toml
[plugins."browser-use@openai-bundled"]
enabled = true

[plugins."documents@openai-primary-runtime"]
enabled = true

[plugins."spreadsheets@openai-primary-runtime"]
enabled = true

[plugins."presentations@openai-primary-runtime"]
enabled = true

[plugins."github@openai-curated"]
enabled = true
```

請 Codex 整理成表格：

```md
| Plugin | 狀態 | 用途 |
|--------|------|------|
| GitHub | 已啟用 / 未啟用 | repo / issue / PR |
| Browser Use | 已啟用 / 未啟用 | 測網頁與截圖 |
| Documents | 已啟用 / 未啟用 | Word |
| Presentations | 已啟用 / 未啟用 | PowerPoint |
| Spreadsheets | 已啟用 / 未啟用 | Excel / CSV |
```

## 步驟三：到 Codex Desktop 啟用缺少的外掛

如果缺少外掛，請使用者到 Codex Desktop app 尋找：

```text
Settings / 設定
Plugins / Marketplace / Apps / Connectors
```

優先啟用：

1. GitHub
2. Browser Use
3. Documents
4. Presentations
5. Spreadsheets

有需要再啟用：

- Gmail
- Google Calendar

> [!warning]
> Gmail / Calendar 涉及個人資料，不是每個人都需要。確定要讓 Codex 協助信件或行程，再連接。

---

## 步驟四：GitHub 要分成兩件事

GitHub 很容易混淆，請分清楚：

| 類型 | 用途 | 檢查方式 |
|------|------|----------|
| GitHub CLI (`gh`) | commit / push / 建 repo / GitHub Pages | `gh auth status` |
| Codex GitHub connector | 對話中讀 repo / issue / PR | 請 Codex 列出 GitHub 帳號與 repo |

檢查 GitHub CLI：

```powershell
gh auth status
```

成功時應看到：

```text
Logged in to github.com account <你的帳號>
Git operations protocol: https
Token scopes: ... repo ...
```

檢查 Codex GitHub connector：

```text
幫我確認 GitHub connector 是否已連接，並列出我可以存取的 GitHub 帳號與幾個 repo。
```

如果出現：

```text
accounts: []
installations: []
repositories: []
```

代表 Codex Desktop 的 GitHub connector 還沒連上。下一集用 [[Codex 懶人包/02-連接-GitHub]] 解決。

## 步驟五：Obsidian 先知道，不急著裝

Obsidian / MCPVault 是第二大腦主題，建議後面再做。

原因：

- 每個人的 vault 位置不同
- 要處理資料夾授權
- 要解釋 `AGENTS.md` 和 Obsidian 駕駛艙的分工
- MCP 裝錯路徑會讓新手卡住

本集只要讓觀眾知道：

> 之後要讓 Codex 讀你的第二大腦，會用 [[Codex 懶人包/03-建立第二大腦-Obsidian]]。

## 新手常見問題

| 問題 | 解法 |
|------|------|
| Skills 跟 Plugins 差在哪 | Skills 是工作流程；Plugins 是外掛能力包 |
| Connector 跟 CLI 差在哪 | Connector 給 Codex 對話讀外部服務；CLI 給本機命令操作 |
| 一定要裝 Gmail / Calendar 嗎 | 不一定，需要整理信件或行程再裝 |
| Obsidian 要本集就裝嗎 | 不建議，留到第二大腦集數 |
| 我看不到某個外掛 | 可能尚未啟用 marketplace，或 Codex 版本不同 |
| 裝越多越好嗎 | 不是，先裝高頻、安全、用得到的 |
