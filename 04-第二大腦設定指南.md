# Codex 懶人包 #04：第二大腦設定指南

> 版本：v0.1（Codex 版）
> 更新日期：2026-04-26
> 前置：已完成 #03（Obsidian 已連 Codex）

---

## 這個懶人包會幫你做什麼？

把空 Obsidian vault 變成完整的 AI 知識管理系統：
- 三層資料夾（輸入 → 消化 → 輸出）
- AGENTS.md 工作規則
- 常用筆記模板
- 每週知識重整排程
- Web Clipper 收集流程

完成後你只要：
1. **平時**：看到好文章 → Web Clipper 存到 Clippings
2. **每週日**：手動觸發或系統排程跑「知識重整」

---

## 階段一：認識你

### 步驟一：詢問使用者

> 🖐️ Codex 依序問：
> 1. 教什麼**科目**？
> 2. 教什麼**年級**？
> 3. 希望用什麼**語言**回答？（預設：繁體中文）
> 4. 其他偏好？

---

## 階段二：建立三層資料夾結構

### 步驟二：建資料夾

```
vault/
├── Clippings/           ← 輸入：網路上看到的好文章、影片筆記
├── 知識庫/              ← 消化：AI 整理後的結構化知識
├── 創作庫/              ← 輸出：你自己的原創內容
├── 每日筆記/            ← 時間管理：每天和每週的紀錄
├── Templates/           ← 模板
└── AGENTS.md            ← Codex 的「班規」
```

| 層 | 資料夾 | 白話 | 誰寫的 |
|---|---|---|---|
| 輸入 | `Clippings/` | 你「剪」下來的東西 | 別人的，你只收集 |
| 消化 | `知識庫/` | AI 幫你整理 | AI 寫，你審 |
| 輸出 | `創作庫/` | 你自己的作品 | 你寫的 |

已存在則跳過。

## 階段三：建立 AGENTS.md

### 步驟三：在 vault 根目錄建 AGENTS.md

> 💡 Codex 每次工作會自動讀取當前目錄的 `AGENTS.md`（同層級或上層）。把它放在 vault 根。

```markdown
# 我的第二大腦 — AGENTS.md

## 關於我
- 我是[科目][年級]老師
- 這個 vault 是我的教學第二大腦

## 語言偏好
- 所有回應請使用[語言]
- 筆記內容也用[語言]

## 筆記庫結構

| 資料夾 | 用途 | 存什麼 |
|---|---|---|
| `Clippings/` | 輸入 | 網路剪藏、影片筆記、別人的資料 |
| `知識庫/` | 消化 | AI 整理後的結構化知識（含 index.md） |
| `創作庫/` | 輸出 | 教案、教材、報告、原創 |
| `每日筆記/` | 時間管理 | 每日紀錄、週計畫、知識重整週報 |
| `Templates/` | 模板 | 各種筆記固定格式 |

## 工作規則

### 新增筆記時
- 一律加 frontmatter（title、date、tags）
- 自動判斷資料夾：別人文章→Clippings/，AI 整理→知識庫/，原創→創作庫/，每日紀錄→每日筆記/

### Clippings 處理
- Web Clipper 剪到的自動進 `Clippings/`，是原始資料**不要修改**
- 每週知識重整時 AI 會把它消化進知識庫

### 知識庫維護
- `index.md`（目錄）和 `log.md`（操作紀錄）
- 每次新增頁面就更新 index
- 每次重整就在 log 加紀錄

### 「幫我新增到筆記」時
- 代表存到本 vault
- 自動判斷資料夾
```

## 階段五：建立知識庫初始檔案

### 步驟五：建 index.md 與 log.md

在 `知識庫/` 建：

```markdown
type: index
updated: [今天日期]
---

# 知識庫索引

> 最後更新：[今天日期]

目前知識庫是空的。當你開始用 Web Clipper 收集文章，每週重整就會把 Clippings 消化成頁面。

| 頁面 | 一行摘要 |
|---|---|
| （等待第一次重整） | |
```

**log.md**：
```markdown
---
title: 知識庫操作紀錄
type: log
---

# 知識庫操作紀錄

## [[今天日期]] 初始化 | 建立第二大腦

**操作類型：** 初始化

**建立的結構：** Clippings/、知識庫/、創作庫/、每日筆記/、Templates/、AGENTS.md
```

## 階段七：每週知識重整排程

### 步驟七：設定排程

> ⚠️ **Codex CLI 沒有原生排程功能**，要靠系統排程器觸發 `codex exec`。

#### macOS / Linux：cron

`crontab -e` 加：

```
17 9 * * 0 cd "[vault路徑]" && codex exec "$(cat ~/.codex/prompts/weekly-review.txt)" >> ~/.codex/weekly-review.log 2>&1
```

並建立 `~/.codex/prompts/weekly-review.txt`，內容見下方「重整 prompt」。

#### Windows：工作排程器

1. 開「工作排程器」→ 建立基本工作
2. 觸發程序：每週日 09:17
3. 動作：啟動程式 `codex`，引數：
   ```
   exec --cwd "G:\我的雲端硬碟\[vault名]" "@~/.codex/prompts/weekly-review.txt"
   ```
   （如果 `@檔案` 語法在你的 codex 版本不支援，把整個 prompt 直接寫進引數，或寫一個 `.bat` 檔包起來）

#### 重整 prompt

```
請執行 Obsidian 筆記庫的每週知識重整：

第一步：盤點本週變動
- 找過去 7 天內 vault 中新增/修改的筆記
- 分三類列：Clippings/（輸入）、創作庫/（原創）、知識庫/（已有頁面）

第二步：消化 Clippings → 知識庫
- 讀本週新增 Clippings 完整內容
- 分析主題、找跨篇關聯
- 為每個主題群產一篇結構化知識庫頁面（frontmatter + 核心概念 + 連結 + 缺口）
- 存到知識庫/
- 沒新 Clippings 跳過

第三步：創作庫回流
- 讀本週創作庫變動
- 有新概念就更新對應知識庫頁面

第四步：知識庫健康檢查
- 連結檢查、知識缺口、孤兒頁面

第五步：更新 index.md 與 log.md

第六步：產週報筆記
- 存到每日筆記/
- 檔名：YYYY-WXX 知識重整.md

完成後通知：「本週知識重整已完成，請查看週報筆記。」
```

> 💡 沒設排程也行，每週日對 Codex 說「跑一次知識重整」就會跑這段。
