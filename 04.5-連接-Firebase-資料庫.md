# Codex 懶人包 #04.5：連接 Firebase 資料庫

> 版本：v0.1（Codex 版）
> 更新日期：2026-04-26

---

## 這個懶人包會幫你做什麼？

讓 Codex CLI 直接操控 Firebase Cloud Firestore：
- 自然語言建集合、CRUD（透過 Firebase MCP 工具）
- 即時更新（學生輸入後展示頁面馬上顯示）
- **千人研習也撐得住**（並發 100 萬）
- **永遠不會閒置暫停**

> 💡 對老師而言這是比 Supabase 更友善的預設選擇。Supabase 適合重度 SQL 統計分析，Firebase 對絕大多數教學工具限制更少。

### Firebase vs Supabase

| 比較項目 | Supabase（#04） | Firebase（本懶人包） |
|---|---|---|
| 資料庫類型 | SQL | NoSQL |
| 免費專案數 | 2 個 | 無限 |
| 閒置暫停 | 一週沒用會暫停 | **不會暫停** |
| 並發連線（免費） | 200 | **100 萬** |
| 即時更新 | 有 | 更強（onSnapshot） |
| MCP 工具 | 完整 | **完整**（list/query/add/update/delete + auth + messaging + storage） |

## 階段一：建立 Firebase 專案

### 步驟零：環境檢查

1. 作業系統 / 網路 / `node --version`（沒裝就裝）
2. `npx --version`

---

### 步驟一：建立 Firebase 專案

> 🖐️ 到 https://console.firebase.google.com → 建立專案：
> - 專案名稱：`my-teaching-tools`
> - Google Analytics 可不啟用
> - 等建立完成

---

### 步驟二：啟用 Cloud Firestore

> 🖐️ Console → Firestore Database → 建立資料庫：
> - Standard 版
> - 位置：`asia-east1 (Taiwan)` 或 `asia-northeast1 (Tokyo)`
> - **安全性規則：「以正式版模式啟動」**（不要選測試模式！測試模式 30 天會過期）
> - 建立

---

### 步驟二之二：設定 Firestore 安全規則（自動化）

請 Codex 在使用者的專案資料夾建立三個檔案：

**1. `firestore.rules`**：
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /wordcloud_words/{document} {
      allow read, write: if true;
    }
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
```

**2. `firebase.json`**：
```json
{
  "firestore": {
    "rules": "firestore.rules"
  }
}
```

**3. `.firebaserc`**：
```json
{
  "projects": {
    "default": "[使用者的 Firebase 專案 ID]"
  }
}
```

部署規則：
```bash
npx -y firebase-tools@latest deploy --only firestore:rules
```

成功會看到 `Deploy complete!`。

> 💡 之後新工具：編輯 `firestore.rules` 加 `match /votes/{document} { allow read, write: if true; }` → 對 Codex 說「部署 Firestore 規則」 → Codex 自動 `firebase deploy --only firestore:rules`。

## 階段二：連接 MCP

### 步驟三：把 Firebase 註冊為 Codex 的 MCP server

**方法 A：用 codex CLI**

```bash
codex mcp add firebase -- npx -y firebase-tools@latest mcp
```

**方法 B：手動編輯 `~/.codex/config.toml`**

```toml
[mcp_servers.firebase]
command = "npx"
args = ["-y", "firebase-tools@latest", "mcp"]
```

---

### 步驟四：登入 Firebase CLI

```bash
npx -y firebase-tools@latest login
```

> 🖐️ 瀏覽器會開 Google 登入頁，授權即可。

驗證：
```bash
npx -y firebase-tools@latest projects:list
```

> ⚠️ `firebase login` 必須在**互動式終端**執行。如果在 Codex 對話中卡住，請使用者打開 cmd / PowerShell 手動跑一次。

---

### 步驟五：重啟 Codex 並驗證

> 🖐️ 完全關閉 Codex CLI 再開。

對 Codex 說「列出我的 Firebase 專案、再列出 Firestore 中的集合」（新專案應為空）。

---

### 步驟六：功能測試

1. 在 Firestore 建集合 `test_collection`，新增一筆 `message`「Firebase 連接測試成功」
2. 查 `test_collection` 確認能讀
3. 刪除測試文件
4. ✅ 「連接測試成功！」

## 如果失敗

對 Codex 說：「Firebase 懶人包失敗，幫我檢查重新處理。」

完全重置：
```bash
codex mcp remove firebase
```

重登：
```bash
npx -y firebase-tools@latest logout
npx -y firebase-tools@latest login
```

## 常見問題

| 問題 | 解法 |
|------|------|
| `npx: command not found` | 確認 Node.js 已裝，重啟終端機 |
| 查詢失敗 | 確認 `firebase login` 已成功 |
| 看不到 Firestore 資料 | 確認 Console 已啟用 Firestore |
| 安全性規則過期 | 用了測試模式。改白名單規則（步驟二之二） |
| `Permission denied` | rules 沒允許這個集合 |
| `Collection id is invalid because it contains /` | `collection_path` 別含尾巴 `/`，寫 `wordcloud_words` 不要 `wordcloud_words/` |
| `firebase login` 在對話中卡住 | 開 cmd/PowerShell 手動跑 `npx firebase-tools login` |
