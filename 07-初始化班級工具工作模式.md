## 這個懶人包會幫你做什麼？

### 一次性建立（5-10 分鐘）

- 「**班級工具總資料夾**」（Google 雲端硬碟內）
- **AGENTS.md** 專案藍圖（含 `.gitignore` 防 token 洩漏）
- **GitHub 公開 repo** + Pages 啟用
- **Obsidian 對應資料夾** + 工作筆記.md（駕駛艙）
- **`shutdown.sh` 收工腳本** + AGENTS.md 觸發規則

| 項目 | Claude Code 版 | Codex 版（本懶人包） |

| 班規檔 | `CLAUDE.md` | `AGENTS.md` |
| 收工機制 | `/收工` Skill（`~/.claude-skills/shutdown/`） | `~/codex-tools/shutdown.sh` + AGENTS.md 規則 |

### 步驟 2：建 AGENTS.md（專案藍圖）+ .gitignore

`AGENTS.md`：

```markdown
# <總資料夾名> — 我的班級工具總專案

## 對話開始時請先讀
進度與最近更動都在 Obsidian：`<vault名>/<總資料夾名>/工作筆記.md`

## 工作模式
- **加新工具**：對 Codex 說「我想做一個 XXX 工具」→ Codex 會建 `tools/<工具名>/` 子資料夾、引導跟著影片做
- **結束工作**：對 Codex 說「**收工**」→ 自動觸發 `~/codex-tools/shutdown.sh` SOP（commit + push + 更新 Obsidian 工作筆記）
- **接續工作**：對 Codex 說「讀工作筆記、告訴我上次做到哪」

## 工作桌 + 三個家
- 📋 GDrive 工作桌：`G:\我的雲端硬碟\<總資料夾名>\`
- 🐙 GitHub repo：`<你的帳號>/<repo名>`（公開）
- 📘 Obsidian 駕駛艙：`<vault名>/<總資料夾名>/工作筆記.md`
- 🔥 Firebase 專案：`my-teaching-tools`

## 「收工」觸發規則（給 Codex 讀的）

當使用者說「收工」、「結束了」、「準備換電腦」、「該同步的同步」、「先到這裡」等收尾語時，**請依照以下 SOP 執行**：

1. 從對話歷史摘要：今天做了什麼（完成檔案、決策、新坑）
2. 找到工作目錄（當前 cwd）和對應的 Obsidian 工作筆記（`<vault>/<同名資料夾>/工作筆記.md`）
3. 更新 Obsidian 工作筆記：
   - 「⏯️ 上次做到哪」段：最後動作、完成檔案、對話脈絡
   - 「🗓️ 最近更動紀錄」表格加一行：今天日期 + 摘要 + ✅✅✅
   - 「🕳️ 踩坑筆記」（若有新坑）
4. Git commit + push：
   ```bash
   cd "<工作目錄>"
   git config windows.appendAtomically false
   git add <今天動到的檔案，不要 add .claude/ 或 .codex/>
   git commit -m "<今天工作摘要>"
   git push origin <branch>
   ```
   - commit 標題：「動詞 + 對象」
   - 正文 3-5 條 bullet 描述變動 + 為什麼
5. 給三勾表格報告同步狀態

不該做：
- ❌ 沒實質進度的對話也跑同步
- ❌ 把 `.claude/`、`.codex/` 進去
- ❌ commit 訊息寫「更新」、「修改」這種沒資訊的字

## 工具清單
（之後加新工具時自動更新）
- （尚無）

## 工作注意事項
- 學生資料一律去識別化（只用座號 + 班級代號）
- commit 訊息寫清楚做了什麼 + 為什麼
- 收工前說「收工」讓 Codex 同步三方
```

## 階段三：建立 shutdown.sh 收工腳本

### 步驟 5：建立腳本

```bash
mkdir -p ~/codex-tools
```

`~/codex-tools/shutdown.sh`：

```bash
#!/bin/bash
# Codex 收工同步腳本
#
# 用法：cd 到工作目錄後執行：
#   bash ~/codex-tools/shutdown.sh "今天工作摘要"
#
# 會做：
#   1. git add 排除 .claude/ .codex/
#   2. git commit 用提供的摘要
#   3. git push
#
# Obsidian 工作筆記更新由 Codex 在對話中完成（透過 mcpvault MCP）

set -e

WORKDIR="${PWD}"
SUMMARY="${1:-收工自動同步}"
LOG="$HOME/codex-tools/shutdown.log"

mkdir -p "$HOME/codex-tools"

log() { echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a "$LOG"; }

log "========== 收工觸發 =========="
log "工作目錄：$WORKDIR"

cd "$WORKDIR"

if [ ! -d ".git" ]; then
    log "  → 不是 git repo，跳過"
    exit 0
fi

# GDrive 內 repo 必要設定
git config windows.appendAtomically false 2>/dev/null || true

# 加入 tracked 的變更，避開 .claude/ .codex/
git add -u
# 新增的檔案逐一檢查（排除 .claude/ .codex/）
for f in $(git ls-files --others --exclude-standard); do
    case "$f" in
        .claude/*|.codex/*) continue ;;
        *) git add "$f" ;;
    esac
done

if git diff --cached --quiet; then
    log "  → 沒有要提交的變更"
    exit 0
fi

git commit -m "$SUMMARY" || { log "  ❌ commit 失敗"; exit 1; }

if git push origin HEAD 2>/dev/null; then
    log "  ✅ commit + push 完成"
else
    log "  ⚠️  commit 完成但 push 失敗（可能離線或 remote 未設）"
fi
```

設可執行（macOS / Linux）：
```bash
chmod +x ~/codex-tools/shutdown.sh
```

> 💡 Windows 上若用 Git Bash 執行，不需 `chmod`，直接 `bash ~/codex-tools/shutdown.sh "..."` 即可。

```markdown
## 收工同步流程

當使用者說「收工」、「結束了」、「準備換電腦」、「該同步的同步」、「先到這裡」等收尾語時，**請依以下步驟執行**：

1. **盤點今天工作**：從對話歷史摘要完成的檔案、決策、踩到的新坑

   ```bash
   bash ~/codex-tools/shutdown.sh "<今天工作摘要的 commit message>"
   ```
4. **回報三勾表格**：

   | 平台 | 動到的檔案 | 狀態 |
   |------|----------|------|
   | GDrive | ... | ✅（自動） |
   | Obsidian | 工作筆記更新 | ✅ |
   | GitHub | commit + push | ✅ |

不該做：
- ❌ 對沒實質進度的對話也跑同步
- ❌ commit 訊息寫「更新」、「修改」這種沒資訊的字
- ❌ 把 `.claude/`、`.codex/` 進去（腳本已自動排除）
```

## FAQ

| 問題 | 解法 |
|---|---|
| `gh repo create` 報錯 | `gh auth login` 重登 |
| GDrive 內 commit 撞 GDrive | 已設 `windows.appendAtomically false`，重試 |
| 「收工」沒觸發 | 確認 `~/.codex/AGENTS.md` 有規則段；或手動說「按收工 SOP 收工」 |
| 換電腦了 | 工作檔 GDrive 自動同步；`git pull`；**重跑這份懶人包**裝 shutdown.sh |
| 工作筆記沒被更新 | 確認 vault 路徑、Obsidian MCP 已連 |
